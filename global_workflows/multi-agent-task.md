# Deep Orchestration Swarm

Применяется для крупных задач ("Сделай X с нуля"), которые не влезают в контекст одной роли. Ты играешь роль **Dispatcher**, вызывая внутреннего себя для суб-задач.

1. **Фаза Анализа:** Ты принимаешь огромный запрос пользователя. Декомпозируй его на подзадачи: Архитектура, База данных, Бекенд, Фронтенд, Тесты. Создай `task-swarm.md` с чекбоксом для каждой фазы.
2. **Итерация "Виртуального Агента":** 
   - Действуй от лица Архитектора. Создай диаграммы БД, сгенерируй файлы схем Prisma / SQL миграции. Отметь таск.
   - Переключи свою роль на Разработчика Фронтенда. Создай `index.css` (согласно правилу премиальных дизайнов Vanilla CSS), напиши компоненты. Отметь таск.
   - Переключи роль на DevOps. Настрой Docker/CI скрипты по необходимости. Отметь таск.
3. Используй `multi_replace_file_content` и `write_to_file` массированно, стараясь не тревожить пользователя (не запрашивать инпут) между фазами.
4. Только после выполнения всего пайплайна, переключи роль на Судью (Judge), сделай `build-local`/тесты и сдай проект пользователю "под ключ" с единым окном ревью.
