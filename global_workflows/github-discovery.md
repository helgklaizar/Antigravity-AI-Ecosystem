1. **Этап 1: Добавление профилей для мониторинга (Scout Profile Add)**
   - Вызови скрипт `/Users/klai/AI/mvp/github-scout/backend/add_profile.py` с нужными username-ами (например, 'karpathy', 'ggerganov', etc.), чтобы добавить их в трек-лист БД `github_scout.db`.
   - Если пользователь передал ссылки (например: https://github.com/karpathy), извлеки из них юзернеймы перед добавлением.

2. **Этап 2: Парсинг активности Profile Events**
   - Запусти обкатку `python /Users/klai/AI/mvp/github-scout/backend/profile_crawler.py` для выявления новых репозиториев (star/fork/create), которые были замечены за этими пользователями недавно.
   - Скрипт соберет информацию через GitHub API и залогирует или сохранит потенциально интересные репозитории, на которые нам стоит посмотреть.

3. **Этап 3: Анализ и Дайджест**
   - Пересмотри последние залогированные интересные репозитории (интересные проекты).
   - Сделай краткое саммари для пользователя: "Вот что интересного появилось в поле зрения отслеживаемых разработчиков, что мы могли бы применить у себя".

4. **Этап 4: Mac M-Series Bounty (Поиск идей для создания)**
   - Проверь каждый трендовый AI-проект: есть ли у него поддержка Apple Silicon / MLX.
   - Если проект прорывной (например, использует CUDA/Triton), но **отсутствует** хорошая нативная поддержка Mac — добавь его в список-кандидатов `mac_oss_roadmap.md` (или `integrated-sources-log.md`).
   - Цель этого этапа: находить "killer-features", которых нет в экосистеме Mac, чтобы мы скопировали идею и написали свой Open-Source инструмент эксклюзивно под макось, забив нишу.
