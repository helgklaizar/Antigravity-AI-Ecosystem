# Context Optimizer & Memory Manager

Процесс автономной компрессии знаний проекта для предотвращения галлюцинаций LLM и сохранения лимитов токенов.

// turbo-all
1. Проинспектируй папку `docs/`, `brain/` и внутренние папки с KI (`<appDataDir>/knowledge/`).
2. Очисти мусор:
   - Удали устаревшие или пустые логи.
   - Найди обрывки завершенных задач и консолидируй их.
3. Сгенерируй **Граф Знаний** проекта: обнови (или создай) файл `knowledge-graph.md`, в котором Mermaid-диаграммой описаны ключевые компоненты, сервисы и файлы проекта, а также их связи.
4. Конвертируй частые решения и паттерны (найденные в коде) в **Knowledge Items (KIs)** и сохрани их с метриками (timestamps, references).
5. Обнови `README.md` или `GEMINI.md` ссылками на последние актуальные KI.
6. Выведи сводный отчет: "Сжато X тысяч токенов логов, создано Y новых KI".
