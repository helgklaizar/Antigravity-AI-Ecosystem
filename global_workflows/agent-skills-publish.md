# /agent-skills-publish

**Цель:** Автоматизированный сбор воркфлоу, AI Engineering паттернов и SKILL.md со всей локальной машины, их структурирование и публикация нового релиза `mac-agent-skills-registry` на GitHub.

## Фаза 1: Агрегация (Сбор)
1. Собрать глобальные ИИ-воркфлоу из целевой директории `~/.gemini/antigravity/global_workflows/`.
2. Извлечь "Knowledge Items" (знания агента) из `~/.gemini/antigravity/knowledge/`.
3. Сохранить все собранные данные в локальный репозиторий проекта (например, `~/Documents/PROJECTS/agent-skills-registry/`).

## Фаза 2: Структурирование и Форматирование
1. **workflows/**: Сохранить все воркфлоу (`*.md` файлы описывающие end-to-end задачи).
2. **knowledge/**: Сохранить архитектурные решения и паттерны (`engineering_patterns.md` и т.д.).
3. **skills/**: (В разработке) Вычленять специфические технические инструкции, снабжая их метаданными (триггерами) в формате `SKILL.md` (в духе Android Skills).

## Фаза 3: Валидация
1. Проверить целостность файлов (отсутствие битых ссылок, проверка наличия `README.md`).
2. Присвоить версию релиза (SemVer), используя `package.json` или формируя changelog.

## Фаза 4: Публикация на GitHub (Release)
1. Выполнить `git add .` и закоммитить изменения сгенерированным Release message.
2. Отправить изменения в удаленный репозиторий (`git push`).
3. Создать релиз через GitHub CLI:
   `gh release create v<VERSION> --title "Agent Skills Update v<VERSION>" --notes "<Сгенерированный лог изменений>"`

*Примечание:* Данный процесс обеспечивает, что "Анти-RAG" мозг агентов на всех машинах актуален и стандартизирован.
