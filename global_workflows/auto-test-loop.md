# Воркфлоу /auto-test-loop

**Описание:** Агрессивный процесс тотального покрытия кодовой базы тестами. Сам находит слепые зоны, ставит нужные либы, пишет тесты, прогоняет их и не останавливается (в цикле), пока все не позеленеет.

## Требования
- Работает автономно (для полного авторан-режима используй аннотацию над скриптами или включи `turbo` для всех шагов).
- Поддерживает рекурсивный фикс (цикл: генерация -> запуск -> ошибка -> исправление).

## Этап 1: Разведка (Discovery)
1. **Анализ стека:** Изучить `package.json` / `pyproject.toml` / `go.mod` (через `view_file` или `run_command`).
2. **Анализ текущего покрытия:** Найти уже существующие тесты (папки `__tests__`, `tests`, суффиксы `*.test.*`). 
3. Запустить существующие тесты в режиме coverage (если фреймворк уже стоит и настроен).

## Этап 2: Сетап окружения (Setup)
1. Если тестового фреймворка нет или не хватает нужных плагинов (моков, coverage-утилит) — подготовить команду установки.
2. Использовать современные решения (Vitest для TS/JS, Pytest для Python, Testify для Go).
3. Создать или обновить базовый конфиг тестов (`vitest.config.ts`, `jest.config.js`, `pytest.ini` и т.д.).
// turbo
`run_command`: установить зависимости (npm install / pip install / go get).

## Этап 3: Итерационное покрытие (The Loop)
Для каждого файла с бизнес-логикой (игнорируя конфиги, типы и "чистый" UI без сложной логики):

1. **Генерация:** Сгенерировать набор тестов для конкретного модуля. Написать моки для БД, сети, сторонних API.
2. **Изолированный запуск:** Запустить тест только для этого файла.
// turbo
`run_command`: прогон тестов выбранного файла (например, `vitest run src/utils.test.ts`).
3. **Лечение (Auto-Fix):** 
   - Если статус упал (красный), получить логи через `command_status`.
   - Внести исправления (либо в тест, если он кривой, либо моки).
   - Повторять запуск до зеленого статуса без вмешательства пользователя.
4. Переходить к следующему файлу только когда текущий полностью зеленый.

## Этап 4: Глобальная проверка (Sanity Check)
1. Когда все файлы покрыты тестами, запустить весь сьют полностью.
// turbo
`run_command`: полный прогон (например, `vitest run` или `pytest`).
2. Если появляются конфликты глобального стейта (тесты мешают друг другу) — найти утечку стейта, исправить (например, добавив `beforeEach`/`afterEach` или `cleanup`).
3. Огласить финальные метрики покрытия и выдать отчет.
