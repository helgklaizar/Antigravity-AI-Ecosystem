0. **СИСТЕМНОЕ:** Строго первым делом выведи в чат: `🔄 **Контекст:** [Агент: devops] | [Скилл: tools] | [Воркфлоу: /github-publish]`

// turbo-all
1. Проведи анализ директории, определи стек, назначение и статус проекта.
2. Проверь наличие и качество файлов `README.md` и `CONTRIBUTING.md`. Если они отсутствуют или базовые — перепиши их.
3. Требования к `README.md` (Premium Aesthetics):
   - Центрированный блок с заголовком и Emoji/Логотипом.
   - Красивые бейджи-щиты (`img.shields.io`) для технологий, лицензий, статуса сборки.
   - Короткое емкое описание (TL;DR).
   - Основные секции: 🚀 Изюминка (Features), 🕹 Быстрый старт, 🏗 Архитектура, 🗺 Roadmap, 🤝 Contributing.
   - Без "скучных стен текста". Структурированно, с таблицами и цитатами.
4. Создай или обнови `.gitignore` (убедись, что скрыты `node_modules/`, `.DS_Store`, `.gemini/`, `.agents/`, `docs/sprints/` — чтобы внутренняя кухня ИИ оставалась локально).
5. Сделай коммит: `chore(docs): premium github formatting`.
6. Если у юзера есть удаленный репозиторий — сделай `git push`. Если нет — скажи: "Репо оформлен, закоммичен. Куда пушим? Скажи ссылку для `git remote add`."
