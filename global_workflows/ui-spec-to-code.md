# Spec-Driven UI Pipeline

**Требования проекта:** Премиальные дизайны, Vanilla CSS (никакого Tailwind без запроса), HSL/динамические палитры, плавные анимации и Glassmorphism.

### Этап 1: Разбор спецификации
1. Пользователь передает описание UI или скриншот дизайна (используй возможность чтения изображений).
2. Выдели дизайн-токены: цвета (сгенерируй HSL палитру), типографику (Modern Google Fonts), тени, радиусы скругления.

### Этап 2: Foundation (Основа)
// turbo
3. Создай или обнови файловую структуру (например `src/index.css`), запиши дизайн-токены как CSS-переменные в `:root`. 
4. Создай базовые утилитарные классы анимаций (hover-эффекты, fade-in, micro-interactions).

### Этап 3: Сборка компонентов
// turbo
5. Сгенерируй React / Vue / Vanilla компоненты, используя строго эти переменные. Избегай инлайн стилей (кроме сложных вычисляемых).
6. Реализуй динамику (интерактивы, состояния ошибок, hover'ы) чтобы интерфейс "ожил".

### Этап 4: Интеграция 
7. Интегрируй компоненты на целевую страницу приложения.
8. Проверь адаптивность и контрастность. Предложи пользователю запустить `build-local` для просмотра результата.
