# 🍏 Agent System Config & Instructions

Этот KI (Knowledge Item) содержит актуальную справку по утилитам и правилам вызовов команд, которые предварительно настроены на хост-машине (macOS) и доступны Агенту (мне).

## 🧰 Системные зависимости и пути (Environment Paths)

Всегда учитывай следующие пути при вызове командных скриптов, чтобы избежать ошибок `command not found`:

1. **GitHub CLI (`gh`)**: 
   - Расположение: `/opt/homebrew/bin/gh`
   - **Авторизация пройдена** под юзером `helgklaizar`. Можно напрямую взаимодействовать с GitHub API.
2. **Python Package Manager (`uv`) / `pip`**:
   - Согласно правилу экосистемы, мы **не** создаем локальные виртуальные окружения `venv` для каждого проекта-папки, чтобы не дублировать тяжелые тензоры (MLX, Torch). Вместо этого пакеты управляются глобально. Если пакета нет — используй установку через системный `python3 -m pip install` или `uv` (когда он будет добавлен в PATH).
3. **C++ / Rust (`cargo`)**:
   - Грузится из `~/.cargo/bin`. Все бинарники компилируются в единую папку `target/` на уровне `MVP/for-mac`, чтобы не плодить 100500 мусорных файлов `target/` для воркспейсов.

## ⌨️ Команды и Workflows (Slash Commands)

Система "Антигравити" поддерживает Slash-команды. При вызове через чат (юзером `/название`), Агент загружает соответствующий `*.md` файл из:
`/Users/k/.gemini/antigravity/global_workflows/`

**Самые частые и важные команды для нашей архитектуры:**
- `/github-publish-mac` — премиально оформляет текущий R&D-проект под Apple Silicon: добавляет бейджи экосистемы в README, генерирует `CONTRIBUTING` и пушит в паблик через `/opt/homebrew/bin/gh repo create`.
- `/mac-port-factory` — автономный запуск конвейера, который берет CUDA-репо, дизайнит MVP под MLX и выводит план задач.
- `/new-project` / `/new-module` — быстрая генерация каркаса для очередной идеи.
- У проекта также существуют QA, DevOps, и Role-playing промпты (`/architect`, `/tester` и т.д.).

## ⚠️ Критические правила для Агента 
1. **Не юзать пустые bash команды:** Агентам строго предписано юзать нативные тулзы (`write_to_file`, `replace_file_content`, `grep_search`) вместо вызовов `cat` / `sed` / `grep` внутри терминала. Терминал (инструмент `run_command`) — только для сборок (build), прогона тестов (pytest/unittest) и вызова системных бинарников (gh, cargo, python).
2. **Apple Silicon Only:** У нас нет NVIDIA GPU. Поэтому вызовы `torch.cuda.is_available()` бессмысленны. Мы используем Apple MLX (Metal) и фреймворки с поддержкой `MPS` (Metal Performance Shaders).
3. **Взаимодействие при разработке:** Если пишем математику для Metal (`.metal`), мы тестируем её через `mx.fast.metal_kernel` JIT, а не компилируем отдельную библиотеку — это удобнее для Python R&D на этапе прототипирования.
