# Skill: Technical Pattern Recognition

Use this skill when analyzing new documentation, articles, or libraries to determine their utility for the current Mac/Agentic ecosystem.

## Evaluation Checklist

### 1. Architectural & Global AI Alignment
- Does it fit the **Flat Directory** topology (Mac AI Ecosystem standard)?
- **Global Impact**: Does it introduce new AI paradigms (e.g., Small Language Models, fine-tuning techniques, RAG-alternatives like LLM Wiki)?
- **Agentic Meta-Efficiency**: Can it improve how agents talk to each other or manage long-term memory?

### 2. Tech Stack Optimization
- **Hardware-Aware**: Does it prefer **MLX/Metal** for local tasks?
- **Speed & Latency**: Is it relevant for real-time systems (TDLib, Sync engines)?

### 3. Agentic Integration (Flexibility Focus)
- **Guidelines vs. Rules**: Avoid adding rigid "Never/Always" rules. Prefer **"Default to X, but justify Y"**.
- **Context Injection**: How does this improve the current session's "Mental Model"?
- **Tooling**: Does it suggest a new MCP server or CLI bridge?

### 4. Productivity / DX
- **Hooks**: Can we automate formatting/testing post-edit?
- **Shortcuts**: Does it offer CLI "power-user" features (like custom spinners or status lines)?

## Actionable Output
When reading a new source, always output:
1. **The "Gem"**: The most valuable single idea.
2. **Context Shift**: How this changes our current understanding of a specific project.
3. **Draft Edit**: A specific change to a config or workflow file to implement the idea.
